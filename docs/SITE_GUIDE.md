# 온세상이마케팅이다 사이트 가이드

이 문서는 온세상이마케팅이다 홈페이지의 **방향성·메시지·톤·구조**를 한 곳에 정리한 콘텐츠 헌장입니다. 코드 구조는 `AGENTS.md`/`CLAUDE.md`가 다루므로, 이 문서는 *무엇을 어떤 톤으로 쓰는가*에 집중합니다.

신규 작가·외주·다음 세션의 Claude는 이 문서만 읽어도 일관된 결을 유지할 수 있어야 합니다.

## 1. 프로젝트 개요

| 항목 | 내용 |
|---|---|
| 회사명 | 온세상이마케팅이다 (온세상이마케팅이다) |
| 사업자등록번호 | 112-31-25690 |
| 주소 | 경기도 화성시 동탄구 동탄중심상가1길 36, 8층 801-117A호 (반송동, 원영빌딩) |
| 대표 | 김태훈 |
| 창립 | 2024 (그로힘 → 온세상이마케팅이다 정식 운영 2026) |
| 도메인 | onketing.kr |

### 1-1. 무엇을 하는 회사인가
변호사·의사·한의사·수의사·노무사·세무사 — **6개 전문직 직군 전용 마케팅 대행사**. 처음 맡은 팀이 컨설팅·기획·발행·보고를 끝까지 책임집니다. 일반 광고와 다른 게임을 한다는 전제 아래, 직군별 검색 의도에 맞춘 콘텐츠를 만듭니다.

### 1-2. 무엇을 하지 않는가
- 6개 직군 외 일반 마케팅
- 노출 수·클릭 수 단위 보고
- 담당자 교체·외주 인력 위탁 운영
- 자동화 보고서

이 *경계 진술*이 톤의 핵심입니다. "다 잘합니다"가 아니라 "이것만 합니다"로 자신감을 표현합니다.

### 1-3. 타겟
- **1차 의사결정자**: 6개 직군의 대표·원장·사장
- **상황**: 마케팅 대행사를 한 번 이상 써봤지만 의뢰로 이어지지 않은 사람
- **고민**: 광고비는 나가는데 의뢰는 늘지 않음. 담당자가 자주 바뀌면서 매번 처음부터 다시 설명한 경험.

### 1-4. 차별점 (3가지)
1. **처음 맡은 팀이 끝까지 갑니다** — 담당자 교체 없이 한 팀이 컨설팅·기획·발행·보고를 책임집니다. 재계약율 90%+로 검증됩니다.
2. **노출이 아닌 의뢰로 평가합니다** — 보고는 노출 수·클릭 수가 아닌 상담 신청 수 단위
3. **6개 직군만 합니다** — 직군 특화 키워드 데이터베이스, 직군 작가 배정

### 1-5. 운영 약속 (1줄)
콘텐츠는 직군별 규정을 점검한 뒤 발행하며, 발행 후 지적이 들어오면 영업일 1일 내로 조치합니다. *(시그니처 카피로 쓰지 않습니다. 강조 대신 기본 약속으로 한 곳에만 짧게 노출합니다.)*

## 3. 톤·매너 가이드 (가장 중요한 섹션)

### 3-1. 4대 원칙

#### 1) 자기 평가형 형용사 금지
| 사용 안 함 | 대신 사용 |
|---|---|
| 가장 잘 만듭니다 | 직군별로 다르게 만듭니다 |
| 최고의 마케팅 | 처음 맡은 팀이 끝까지 가는 마케팅 |
| 자연스럽게 알립니다 | 콘텐츠로 쌓습니다 |
| 완벽 준수 | 사전 점검 |
| 혁신적인 솔루션 | (사실 진술로 대체) |
| 가장 빠른 길 | 가장 먼저 만나는 자리 |
| 단 하나의 마케팅 팀 | 한 팀이 끝까지 보는 마케팅 팀 |
| 퍼스널 브랜드 구축 | 전문가 채널을 키웁니다 |

자기 평가는 검증이 어렵습니다. 검증 가능한 사실(연차·직군 수·고유명사·재계약율·사업자 정보)로 자신감을 표현합니다.

#### 2) 한 문장 한 의미
"그리고·또한·또는"으로 두 의미를 묶지 않습니다. 짧게 끊습니다.

```
❌ 직군 키워드를 분석하고 검색 의도에 맞춘 콘텐츠를 만들고 보고는 상담 신청 수로 합니다.
✅ 직군 키워드를 분석합니다. 검색 의도에 맞춘 콘텐츠를 만듭니다. 보고는 상담 신청 수로 합니다.
```

#### 3) 단정형 마침
"~할 수 있습니다·되는 편입니다"가 아니라 "~합니다·됩니다"로 끝냅니다.

```
❌ 검색 상위 노출을 끌어낼 수 있습니다.
✅ 검색 상위 노출까지 책임집니다.
```

#### 4) 검증 어려운 일반화 금지
"대행사 절반은", "단 하나의", "모든" 같은 표현은 검증할 수 있을 때만 씁니다.

```
❌ 대행사 절반은 노출 수만 보고합니다.
✅ 대부분의 대행사는 노출 수만 보고합니다.
```

### 3-2. 자신감 표현 방식

자기 평가가 아닌 **사실 진술**로 자신감을 표현합니다.

| 자신감의 출처 | 표현 |
|---|---|
| 직군 수 | "6개 직군만 합니다" |
| 운영 연속성 | "재계약율 90%+ / 처음 맡은 팀이 끝까지" |
| 사업자 정보 | "112-31-25690 / 화성시 동탄…" |
| 정부 인증 | "2024 중기부 예비창업 우수 기업 / G스타트업 최우수" |
| 졸업·재직 | "한신대 정보통신학부 / 'T'사 퍼포먼스 마케터" |
| 입주 | "2025 중앙대 캠퍼스타운" |

### 3-3. 약점 인정 1줄 (권장)
"규모는 작습니다" — 규모를 숨기지 않고 인정한 뒤 *그래서 모든 의뢰를 직접 봅니다* 로 강점으로 전환합니다. 신생 회사·소규모 팀이 자신감을 잃지 않는 가장 효과적인 표현 방식입니다.

### 3-4. 압박 없는 마침
- "판단은 직접 하시면 됩니다"
- "계약 압박 없음"
- "첫 상담 무료"

전문직 의사결정자는 압박을 가장 싫어합니다. 압박 없는 자신감이 신뢰를 만듭니다.

### 3-5. AI 티 회피 체크리스트 (5문항)
콘텐츠 발행 전 5개 질문에 모두 *예* 라고 답할 수 있어야 합니다.

- [ ] 추상적 형용사가 0개인가? ("최고의·완벽한·자연스럽게·혁신적인·놀라운·압도적인")
- [ ] 한 문장이 한 의미만 담고 있는가?
- [ ] 단정형으로 끝나는가? ("~합니다·됩니다")
- [ ] "그리고·또한·또는" 연결어가 최소화됐는가?
- [ ] 검증 가능한 고유명사·숫자·수치가 1개 이상 들어 있는가?

### 3-6. 금지어 리스트
다음 단어가 등장하면 다시 봅니다.

```
가장 / 최고의 / 최선의 / 완벽한 / 자연스럽게 / 혁신적인 /
단 하나의 / 모든 / 누구나 / 압도적인 / 놀라운 / 차별화된 /
원스톱 / 토탈 솔루션 / 퍼스널 브랜드 / ROAS만 보장 /
효율적인 / 효과적인 (단독 사용)
```

### 3-7. 권장 어휘
```
처음 맡은 팀이 끝까지 갑니다 / 한 사람이 봅니다 / 끝까지 봅니다 /
단정합니다 / 발행합니다 / 의뢰로 이어집니다 /
판단은 직접 하시면 됩니다 / 계약 압박 없음 /
영업일 1일 내 회신 / 첫 상담 무료
```

## 4. 디자인 톤

### 4-1. 컬러
| 용도 | 값 | 비고 |
|---|---|---|
| 강조 1 (그린) | `#58d68d` | CTA, eyebrow, 아이콘 배경, 링크 호버 |
| 다크 섹션 navy | `#0b1220` / `#0f172a` | WhyUsBold, FinalCTA, Footer |
| 본문 | `text-foreground` (slate-900) | 본문 헤딩 |
| 보조 텍스트 | `text-muted-foreground` (slate-600) | 본문 서술 |
| 배경 분리 | `bg-slate-50` / `bg-slate-100` | 섹션 교차 배경 |

**그린 강조색은 절제**합니다. CTA·eyebrow·핵심 강조 외에는 사용을 줄여 등장할 때마다 무게감을 갖게 합니다.

### 4-2. 시각 리듬
밝음 연속 → 다크 앵커 패턴.

```
메인 페이지 12개 섹션:
밝음 → 밝음 → 밝음 → 다크(WhyUsBold) → 밝음 → 밝음 →
밝음 → 밝음 → 밝음 → 밝음 → 밝음 → 다크(FinalCTA)
```

다크 섹션은 *차별점 진술* 과 *최종 CTA* 두 곳에만 박힙니다. 다크가 너무 자주 나오면 앵커 효과가 사라집니다.

### 4-3. 타이포그래피
- **Eyebrow**: `font-semibold text-[#58d68d] text-sm uppercase tracking-[0.25em]` — 영문 라벨
- **H1·H2**: `font-bold tracking-tight` 굵고 단정
- **본문**: `leading-relaxed` 가독성 우선
- **gradient-text**: 그린 그라데이션 텍스트 (강조용)

### 4-4. 컴포넌트 패턴
페이지 단위는 다음 4개 컴포넌트로 구성됩니다.

| 컴포넌트 | 역할 |
|---|---|
| `PageHero` | 페이지 상단 후크 (eyebrow / title+highlight / sub / CTA 2개) |
| `SectionHeading` | 섹션 헤더 (eyebrow / title / sub) |
| `Reveal` | 스크롤 페이드인 애니메이션 래퍼 |
| `CTACard` | 페이지 하단 행동 유도 카드 |

신규 섹션을 만들 때 이 4개를 그대로 사용합니다. 자체 헤더 마크업을 새로 짜지 않습니다.

```
(기타: sitemap.xml, robots.txt, manifest.webmanifest, opengraph-image)
```

### 5-2. 글로벌 네비게이션
헤더 메뉴 5개 + CTA 2개:

```
회사소개 / 마케팅 ▾ / 광고 규정 / 팀 / FAQ      [회사소개서] [무료 상담 신청]
```

`마케팅` 드롭다운은 전문직(대표) / 블로그 / 숏폼 3개.

### 5-3. 동선 의도
사용자는 보통 다음 흐름으로 신뢰를 쌓습니다.

```
메인 → (공감)PainPoints → (차별점)WhyUsBold → (운영 사례)Cases
     → 회사소개(About → CEO 인사말 → 경력)
     → 팀(누가 만드는가)
     → 서비스(블로그/숏폼/전문직)
     → 문의
```

각 페이지는 이 흐름의 *어느 단계*인지 인지하고 카피를 작성합니다.

1. **의도 정의** — 어느 동선의 어느 단계인지 (공감 / 차별점 / 증거 / 행동)

### 9-2. 카피 수정 시
- `git grep` 으로 금지어 등장 빈도 점검

- 카피 수정 후 빌드 → dev 점검 두 단계 모두 거침

### 9-3. 빠른 톤 점검 명령
```sh
git grep -E '가장|최고의|완벽한|자연스럽게|혁신적인|단 하나의|퍼스널 브랜드' src/
```
